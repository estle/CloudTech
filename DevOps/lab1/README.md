# Выполнение лабораторных работ

**UTM**

UTM — программное обеспечение виртуализации, предназначенное для компьютеров ARM64 операционной системы MacOS. Позволяет пользователю установить одну или более виртуальных машин на один физический компьютер и запускать их параллельно с ним.

Под капотом UTM находится QEMU, существующее десятилетиями бесплатное программное обеспечение для эмуляции с открытым исходным кодом, которое широко используется и активно поддерживается.

**Использующиеся операционки**

Для выполнения лабораторной работы будут использоваться следующие компьютеры и операционные системы:
- MacOS Sonoma 14.0, как компьютер А;
- Debian 11 XFCE на виртуалке, как компьютер Б;
- ArchLinux ARM на виртуалке, как компьютер С.

## Лабораторная работа 1

Узнаем ip на компьютерах Б и С командой ```ifconfig``` или ```hostname -I```.

Для удобства представления составим таблицу компьютеров в сети с их IP адресами.

| ПК | ОС ПК | IP адрес |
|:--:|:-----:|:--------:|
|А|macOS 14|-|
|Б|Debian 11|192.168.64.3|
|С|ArchLinux|192.168.64.2|

Убедимся, что компьютеры Б и С пингуются с компьютера А.

**openssh-server package**

Для установки соединения будем удалённо подключаться к компютеру Б на компютере А через протокол ssh, предварительно скачав и запустив на Б и С пакет openssh командами 
```
apt-get install openssh-server
systemctl enable ssh --now
```

**Создаем файл _message.txt_**

На компютере Б в домашней директории создадим файл _message.txt_, в котором будет некоторая цитата от коровы ```fortune | cowsay```
